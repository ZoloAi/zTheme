Icons

SVG icon sprite system for zTheme applications

Understanding the Icon System

zTheme uses an SVG sprite system - a modern, performant approach to icons that:

  • Zero HTTP requests - All icons loaded once, inline with HTML
  • Scalable - Vector graphics look perfect at any size
  • Themeable - Icons inherit text color via fill="currentColor"
  • Customizable - Easy to add, modify, or remove icons
  • Accessible - Full control over ARIA attributes

üí° Why not Font Awesome or Bootstrap Icons? External icon libraries add HTTP requests, bloat your bundle, and include hundreds of icons you'll never use. The sprite system gives you complete control with better performance.

Basic Usage

Including Icons in Your Page (Offline)

To use icons in your HTML page, copy the SVG sprite from dist/icons.svg and paste it right after your opening <body> tag: 

<body>
  <!-- SVG Icon Sprite (from dist/icons.svg) -->
  <svg xmlns="http://www.w3.org/2000/svg" style="display: none;">
    <symbol id="icon-house" viewBox="0 0 16 16" fill="currentColor">
      <!-- icon paths... -->
    </symbol>
    <!-- more icons... -->
  </svg>

  <!-- Your content -->
</body>

Using Icons

Once the sprite is included, reference icons by their id:

Home 

<svg class="zIcon" width="24" height="24">
  <use xlink:href="#icon-house"></use>
</svg>

Adding New Icons

To add new icons to the sprite system:

Step 1: Find or Create SVG

Get SVG icons from:

  • Bootstrap Icons - Open source, MIT licensed
  • Heroicons - Free, MIT licensed
  • Feather Icons - Simple, open source
  • Custom design - Create in Figma, Illustrator, etc.

Step 2: Convert to Symbol

<!-- Original SVG -->
<svg viewBox="0 0 16 16">
  <path d="M8 0a8 8 0 1 0 0 16A8 8 0 0 0 8 0z"/>
</svg>

<!-- Convert to symbol -->
<symbol id="icon-circle" viewBox="0 0 16 16" fill="currentColor">
  <path d="M8 0a8 8 0 1 0 0 16A8 8 0 0 0 8 0z"/>
</symbol>

Step 3: Add to Sprite File

Edit dist/icons.svg and add your new symbol inside the main <svg> container. Then inline it in your HTML page after the <body> tag.

Step 4: Use Immediately

<svg class="zIcon">
  <use xlink:href="#icon-circle"></use>
</svg>

Best Practices

‚úÖ Do

  • Use consistent icon sizing across similar UI elements
  • Ensure icons have proper contrast against backgrounds (WCAG AA minimum)
  • Use .zAlign-middle for inline icons next to text
  • Add aria-hidden="true" for purely decorative icons
  • Keep the sprite file organized with clear comments
  • Test icons at multiple sizes to ensure clarity

‚ùå Don't

  • Mix different icon styles (outlined vs filled) inconsistently
  • Use tiny icons (<12px) - they become unclear
  • Forget to test in high contrast mode
  • Add unused icons to the sprite (keep it lean)
  • Override fill directly - use color utilities instead
  • Use raster images when SVG works better

Accessibility

Decorative Icons

Icons that are purely decorative (next to text labels) should be hidden from screen readers: 

<button class="zBtn zBtn-primary">
  <svg class="zIcon" aria-hidden="true">
    <use xlink:href="#icon-house"></use>
  </svg>
  Home
</button>

Standalone Icons

If an icon is the ONLY indicator (no text), add a label: 

<button class="zBtn zBtn-primary" aria-label="Go to home page">
  <svg class="zIcon">
    <use xlink:href="#icon-house"></use>
  </svg>
</button>

Interactive Icons

For clickable icons, ensure proper focus states and size (minimum 44x44px touch target): 

<button 
  class="zBtn" 
  aria-label="Refresh data"
  style="padding: 0.75rem;">
  <svg class="zIcon zIcon-lg">
    <use xlink:href="#icon-arrow-repeat"></use>
  </svg>
</button>
